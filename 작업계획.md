## Phase 1 — 데이터 크롤링

### 확인된 URL 패턴
- 목록 페이지: `https://www.nl.go.kr/NL/contents/N30501000000.do?pageNum=1&pageSize=100`
- 상세 페이지: `https://www.nl.go.kr/NL/contents/N30501000000.do?schM=view&ackRecKey={ackRecKey}`

### 수집 대상 (총 5,910건 / 591페이지)
| 필드 | 설명 |
|------|------|
| question | 질문 전문 |
| answer | 답변 전문 |
| library | 답변 도서관명 |
| category | KDC 대분류 카테고리 |
| reg_date | 등록일 |
| answer_date | 답변일 |
| views | 조회수 |
| ack_rec_key | 고유 키 (URL 파라미터) |
| source_url | 원문 링크 |

### 크롤링 방법
- 도구: `requests + BeautifulSoup` (JS 렌더링 불필요, 단순 GET 요청으로 접근 가능)
- 순서: 목록 페이지 전체 순회 → ackRecKey 수집 → 상세 페이지 개별 수집
- 요청 간격: 1~2초 (서버 부하 배려)
- 수집 소요 예상: 약 2~3시간 (5,910건 × 1.5초)

### 주의사항
- robots.txt 확인 후 진행
- User-Agent에 학술 목적 명시
- 중간 저장 구현 (중단 시 이어서 수집 가능하도록)

## Phase 2 — DB 저장 + 임베딩 생성

### DB 선택: PostgreSQL + pgvector
```sql
CREATE TABLE qa_records (
    id           SERIAL PRIMARY KEY,
    ack_rec_key  VARCHAR(30) UNIQUE NOT NULL,
    question     TEXT NOT NULL,
    answer       TEXT NOT NULL,
    library      VARCHAR(100),
    category     VARCHAR(100),
    reg_date     DATE,
    answer_date  DATE,
    views        INTEGER,
    source_url   TEXT,
    q_embedding  VECTOR(1536),
    a_embedding  VECTOR(1536),
    created_at   TIMESTAMP DEFAULT NOW()
);

-- 키워드 검색용 전문 인덱스
CREATE INDEX ON qa_records USING GIN (
    to_tsvector('simple', question || ' ' || answer)
);

-- 벡터 검색용 인덱스
CREATE INDEX ON qa_records USING ivfflat (q_embedding vector_cosine_ops)
    WITH (lists = 100);
```

### 임베딩 모델 선택지
| 모델 | 비용 | 한국어 품질 | 권장 |
|------|------|-----------|------|
| OpenAI text-embedding-3-small | 유료(저렴) | 우수 | 권장 |
| multilingual-e5-large | 무료(로컬) | 양호 | 비용 절감 시 |

## Phase 3 — 하이브리드 검색 구현

### 검색 비중: 키워드 35% + 벡터 65%

#### 근거
- 도서관 Q&A 질의는 대부분 설명적 자연어 → 벡터 검색 우위
- 책 제목·저자·ISBN 등 고유명사 포함 시 → 키워드 검색 보완

#### 구현 방법: RRF (Reciprocal Rank Fusion)
```python
def hybrid_search(query, top_k=5, alpha=0.65):
    # alpha = 벡터 비중
    keyword_results = bm25_search(query)   # BM25 키워드 검색
    vector_results  = vector_search(query) # cosine 유사도 벡터 검색

    rrf_scores = {}
    for rank, doc in enumerate(keyword_results):
        rrf_scores[doc.id] = rrf_scores.get(doc.id, 0) + (1 - alpha) / (60 + rank)
    for rank, doc in enumerate(vector_results):
        rrf_scores[doc.id] = rrf_scores.get(doc.id, 0) + alpha / (60 + rank)

    return sorted(rrf_scores, key=lambda x: rrf_scores[x], reverse=True)[:top_k]
```

#### 질의 유형별 alpha 조정 (선택 구현)
| 질의 유형 | alpha 값 |
|-----------|---------|
| 일반 자연어 질문 | 0.65 |
| 책 제목/저자 포함 | 0.30 |
| 주제 탐색형 | 0.80 |

## Phase 4 — Claude API 연계 (RAG)

### 핵심 원칙: 헐루시네이션 배제
- Claude는 검색 결과를 **요약·정리**만 함 (독자적 지식 사용 금지)
- 검색 결과 없으면 "관련 자료를 찾지 못했습니다" 반환
- 출처 URL 의무 표시

### 프롬프트 설계
```
[시스템]
당신은 국립중앙도서관 사서입니다.
반드시 아래 [검색결과]에 있는 내용만을 근거로 답변하십시오.
검색결과에 없는 내용은 절대 추가하지 마십시오.

[이용자 질문]
{user_query}

[검색결과 - 실제 사서 답변 DB]
1. Q: {q1} / A: {a1} / 출처: {url1}
2. Q: {q2} / A: {a2} / 출처: {url2}
3. Q: {q3} / A: {a3} / 출처: {url3}

위 검색결과를 바탕으로 친절하고 명확하게 답변하고,
참고한 원문의 출처 URL을 반드시 표시하십시오.
```

### 모델: claude-sonnet-4-6
